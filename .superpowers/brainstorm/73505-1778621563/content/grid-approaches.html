Grid Layout: How to Center the Last Row

The main design question — how the CSS handles fewer items in the final row

A

CSS Grid + Flexbox Hybrid

Recommended. Use CSS Grid (auto-fill + minmax()) for the full rows, then switch the grid container to flex-wrap: wrap with justify-content: center. This is the simplest way to get centered last rows with fluid column counts.

.card-grid {
  display: flex;
  flex-wrap: wrap;
  justify-content: center;
  gap: 1rem;
}

.card-grid > * {
  flex: 0 1 var(--card-max-width);
  min-width: var(--card-min-width);
}

Pro: Pure CSS, centered last row by default, no JS needed.
Con: Cards in the last row can be slightly wider than others if there's leftover space (flex-grow can be tuned to prevent this).

B

Pure CSS Grid + Auto Margins

Use CSS Grid with auto-fill + minmax(). For centering the last row, add invisible placeholder items or use justify-content: center on the grid itself.

.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(var(--min), 1fr));
  gap: 1rem;
}

Pro: Clean CSS Grid semantics.
Con: CSS Grid's auto-fill left-aligns the last row by default. justify-content: center on a grid centers all rows, which adds gaps when items don't fill the row — the columns don't stretch to fill the container width. There's no pure-CSS-Grid way to "stretch full rows but center the last row."

C

Bootstrap Row/Col (Current Approach)

Keep using Bootstrap's row-cols-N grid system, but add a JS snippet or Hugo logic to center the last row.

Pro: Consistent with existing partials — minimal CSS changes.
Con: Fixed column count (not fluid), centering the last row requires JS or wrapper tricks, and you lose the min/max-based responsive behavior you asked for.
