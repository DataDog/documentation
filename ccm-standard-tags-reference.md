# Part 1: How CCM Tags Work

## 1. Tag Sources, Priority & Normalization

### Where Tags Come From

CCM collects and enriches cost data with tags from multiple sources:

| Source Category | Tag Source | Description |
|----------------|-----------|-------------|
| **All Providers** | Bill columns | AWS CUR columns, Azure cost export columns, GCP billing export columns, OCI FOCUS export columns |
| **Datadog Enrichment** | Host Agent | Tags added to host metadata by the Datadog Agent |
| **Datadog Enrichment** | Software Catalog | Tags associated with a service in the APM Service Catalog |
| **Datadog Enrichment** | Integration Tiles | Tags added to the Datadog integration tile for a cloud account |
| **Datadog Enrichment** | Data Observability | Tags from Datadog Data Observability (powers BigQuery cost allocation) |
| **Datadog Enrichment** | Cloud Network Monitoring | Source and destination dimensions from Cloud Network Monitoring |
| **Kubernetes** | Node / Pod / PV / PVC | User-defined tags on Kubernetes resources monitored with Datadog |
| **Cloud Cost Management** | Cloud Cost Aliases | Friendly-named tags derived from provider bill columns (e.g., `aws_product`) |
| **Cloud Cost Management** | Cloud Cost Allocation | Tags created during cost allocation (e.g., `allocated_spend_type`) |
| **Cloud Cost Management** | FOCUS | Provider-agnostic tags compliant with FinOps FOCUS specification |
| **Tag Pipelines** | User-defined rules | Tags created by applying Tag Pipeline rules to cost data |
| **Custom Allocation Rules** | User-defined rules | Tags created by applying Custom Allocation Rules |

#### Provider-Specific Tag Sources

| Provider | Tag Source | Description |
|----------|-----------|-------------|
| **AWS** | Cost Allocation Tags | User-defined tags in AWS Cost Allocation shown in the CUR |
| **AWS** | Resource Groups Tagging API | User-defined tags on AWS resources via the Groups Tagging API |
| **AWS** | AWS Organizations | Tags on organizational units and accounts |
| **Amazon ECS** | ECS Task / Container | User-defined tags on ECS task definitions and containers |
| **Azure** | User Resource Tags | User-defined tags on Azure resources (from Tags column) |
| **Azure** | System Resource Tags | Azure-defined tags (from AdditionalInfo column) |
| **GCP** | Labels & Tags | Project labels, system labels, user labels, user tags, GKE pod labels |
| **OCI** | User Resource Tags | User-defined tags on OCI resources (from FOCUS cost export Tags column) |
| **Datadog** | Usage Attribution | Up to 3 user-defined tags configured in Usage Attribution (Plan & Usage) |
| **Custom Costs** | Cost File Tags | Tags from uploaded custom cost files |

### Tag Priority (Conflict Resolution)

When multiple sources provide conflicting values for the same tag key, the following priority determines which value wins:

1. **Custom Allocation Rules** *(highest priority)*
2. **FOCUS tags**
3. **Service Catalog**
4. **Amazon ECS Container**
5. **Amazon ECS Task**
6. **Kubernetes Pod**
7. **Kubernetes Persistent Volume**
8. **Kubernetes Node**
9. **Host Agent** *(lowest priority)*

**Important:**
- Bill columns and FOCUS columns are **reserved** and cannot be overridden by any source
- Tag Pipelines **add new tags only** — they do not override existing tags

### Tag Normalization Rules

CCM has different normalization rules from standard Datadog metrics:

| | Keys | Values |
|--|------|--------|
| **Casing** | Lowercased | **Preserves original casing** (e.g., `Machine Learning`) |
| **Special chars** | Replaced with `_` | Kept (letters, marks, numbers, punctuation, symbols) |
| **Leading chars** | Non-letter chars dropped | N/A |
| **Whitespace** | Replaced with `_` | Consecutive whitespace → single space |
| **Max length** | 5,000 chars | 5,000 chars |

An optional **Tag Normalization toggle** in Tag Pipelines settings normalizes values to match standard Datadog metrics normalization (lowercase, etc.).

### Cost Types

| Cost Type | Description |
|-----------|-------------|
| **On-Demand** | Public list price; excludes savings plans, reservations, discounts, taxes, and fees |
| **Amortized** | Distributes commitment savings over the discount term (accrual basis) |
| **Unblended** | Charges shown on the date incurred (cost basis) |
| **Net Amortized** | Amortized cost plus negotiated discounts |
| **Net Unblended** | Unblended cost plus negotiated discounts |

## 2. FOCUS — The Cross-Provider Standard

FOCUS (FinOps Open Cost and Usage Specification) tags are provider-agnostic tags added to **all** cost data — cloud providers, SaaS integrations, Datadog costs, and custom costs. They allow querying similar concepts across every cost source using a single tag name.

### 2.1 FOCUS Tag Definitions (17 tags)

| Tag | Description |
|-----|-------------|
| `providername` | The cloud provider or cost source name (e.g., `aws`, `azure`, `gcp`, `oci`, `datadog`, `Snowflake`) |
| `publishername` | Entity making resources or services available for purchase |
| `servicename` | An offering that can be purchased from a provider |
| `billingaccountid` | Identifier assigned to a billing account by the provider |
| `billingaccountname` | Display name assigned to a billing account |
| `subaccountid` | ID assigned to a grouping of resources or services |
| `subaccountname` | Name assigned to a grouping of resources or services |
| `regionname` | Name of an isolated geographic area where a resource is provisioned |
| `regionid` | Provider-assigned identifier for a region |
| `availabilityzone` | Provider-assigned identifier for a physically separated area within a region |
| `pricingunit` | Provider-specified measurement unit for determining unit prices |
| `billingcurrency` | Currency in which a cloud bill was paid |
| `chargecategory` | Highest-level classification of a charge (see values below) |
| `chargefrequency` | How often a charge will occur (see values below) |
| `chargedescription` | Description of a charge |
| `skumeter` | Additional provider-specific SKU detail |
| `invoiceissuer` | Entity that issued the invoice |

|-------|-------------|

| `adjustment` | Adjustments, rounding errors, or negations |
