## 개요 {#overview}

<div class="alert alert-info">쿼리는 UTC 시간으로 실행되지만, 쿼리의 시간 프레임은 <strong>브라우저의 시간대</strong>에 따라 선택됩니다. 대시보드 구성 메뉴에서 기본 시간대와 UTC 간 표시를 전환할 수 있습니다. 자세한 내용은 <a href="/dashboards/configure/#configuration-actions">대시보드 구성 설명서</a>를 참조하세요.</div>

Datadog의 많은 보기에서는 특정 시간 프레임으로 범위를 지정할 수 있습니다. 시간 제어에는 일반적으로 사용되는 시간 프레임 목록과 빠른 선택을 위한 달력 선택기가 포함됩니다.

시간 프레임은 다음과 같이 구분됩니다.

- **슬라이딩**: 시작 시점과 종료 시점이 시간 경과에 따라 함께 이동합니다(예: `5h`은 항상 최근 5시간을 표시).
- **증가형**: 시작 시점은 고정되고 종료 시점은 "현재(now)"를 따라 이동합니다(예: `since Jun 1`은 6월 1일부터 현재 시점까지의 모든 데이터를 표시).
- **고정형**: 시작 시점과 종료 시점이 모두 특정 시점에 고정됩니다(예: `Jan 1 - Jan 2`).

월, 일, 연도, 시간 또는 분 단위로 증감하려면 시간 프레임의 일부를 선택한 후 `[↑]` 및 `[↓]` 키를 사용하세요.

## 지원되는 구문 {#supported-syntaxes}

### 고정 날짜 {#fixed-dates}

{{< img src="dashboards/guide/custom_time_frames/custom_fixed_time_frame.mp4" alt="사용자 지정 고정 시간 프레임 입력." video="true" width="500" >}}

| 형식                       | 예시                                         |
| ---------------------------- | ------------------------------------------------ |
| `{MMM/MMMM} D`               | 1월 1일<br>1월 1일                               |
| `M/D`                        | 1&#8203;/&#8203;1                                |
| `M-D`                        | 1-1                                              |
| `M/D/{YY/YYYY}`              | 1/1/19<br>1/1/2019                               |
| `M-D-{YY/YYYY}`              | 1-1-19<br>1-1-2019                               |
| `{MMM/MMMM} D, h:mm a`       | 1월 1일, 오후 1:00<br>1월 1일, 오후 1:00             |
| `{MMM/MMMM} D, YYYY, h:mm a` | 2019년 1월 1일, 오후 1:00<br>2019년 1월 1일, 오후 1:00 |
| `h:mm a`                     | 오후 1:00                                          |
| Unix 초 타임스탬프        | 1577883600                                       |
| Unix 밀리초 타임스탬프  | 1577883600000                                    |

모든 고정 날짜는 범위의 일부로 입력할 수 있습니다. 예를 들면 다음과 같습니다.

- `1577883600 - 1578009540`
- `Jan 1 - Jan 2`
- `6:00 am - 1:00 pm`

### 상대 날짜 {#relative-dates}

상대 날짜는 시간이 지남에 따라 자동으로 업데이트되며 현재 시점을 기준으로 계산됩니다.

{{< img src="dashboards/guide/custom_time_frames/custom_relative_time_frame.mp4" alt="사용자 지정 상대 시간 프레임 입력" video="true" width="500" >}}

| 형식                                             | 설명                                                                                                                                                                                                                                                                                          |
| -------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `N{unit}` | 지난 N개 단위의 슬라이딩 창을 표시합니다. 사용 가능한 단위는 아래에 나열되어 있습니다. 시작 시점과 종료 시점이 모두 시간에 따라 이동합니다. 예를 들어 **3 mo**는 지난 3개월을 의미합니다. 종료 시점을 “현재”로 유지하는 증가형 창을 만들려면 `to now` 구문을 사용하세요(예: `10am to now`). 자세한 내용은 [증가형 시간 프레임](#growing-time-frames)을 참조하세요. |

상대 날짜에서 `{unit}`에 사용할 수 있는 문자열은 다음과 같습니다.

- 분: `m`, `min`, `mins`, `minute`, `minutes`
- 시간: `h`, `hr`, `hrs`, `hour`, `hours`
- 일: `d`, `day`, `days`
- 주: `w`, `week`, `weeks`
- 월: `mo`, `mos`, `mon`, `mons`, `month`, `months`

### 증가형 시간 프레임 {#growing-time-frames}

증가형 시간 프레임은 시작 시점이 고정되며 종료 시점은 자동으로 현재(“now”)까지 확장됩니다. 특정 시점 이후의 모든 활동을 확인하려는 경우 유용합니다.

{{< img src="dashboards/guide/custom_time_frames/custom_growing_time_frame.mp4" alt="사용자 지정 증가형 시간 프레임 입력" video="true" width="500" >}}

| 형식          | 설명                                                                                                 |
| --------------- | ----------------------------------------------------------------------------------------------------------- |
| `{date} to now` | `{date}`부터 현재까지의 증가형 기간. 예: **Jan 1 to now**.                                    |
| `{date} - now`  | `{date} to now`와 동일. 예: **Jan 1 - now**.                                                |
| `since {date}`  | `{date} to now`와 동일. 예: **since Jun 1**, **since 5h**, **since 1549116000**.            |
| `from {date}`   | `{date} to now`와 동일. 예: **from Jun 1**, **from 5h**.                                    |

두 형식 모두에서 `{date}`에는 다음 중 하나를 사용할 수 있습니다.

- 상대 축약 표현(예: `5h`, `2d`, `3w`)
- 고정 날짜(예: `Jun 1`, `Feb 2 2pm`, `1/15/2024`)
- Unix 초 또는 밀리초 타임스탬프(예: `1549116000`, `1549116000000`)

### 달력 기준 날짜 {#calendar-aligned-dates}

달력 기준 날짜는 자정, 주 시작일, 월 시작일과 같은 달력 경계에 맞춰 정렬되며 시간이 지나면서 자동으로 업데이트됩니다.

| 형식            | 설명                                                                           |
| ----------------- | ------------------------------------------------------------------------------------- |
| `today`           | 현재 달력 날짜의 시작부터 현재 시점까지                                                |
| `yesterday`       | 이전 달력 날짜 전체                                                        |
| `week to date`    | 월요일 오전 12시부터 현재 시점까지의 현재 주|
| `month to date`   | 1일부터 현재 시점까지의 현재 월                                          |
| `year to date`    | 1월 1일부터 현재 시점까지의 현재 연도                                         |
| `this {unit}`     | 현재 달력 단위부터 현재 시점까지. 예: **이번 달**, **올해**   |
| `last {unit}`     | 이전 달력 단위 전체. 예: **지난달**, **작년**           |
| `previous {unit}` | `last {unit}`과 동일. 예: **지난주**, **지난달**       |
| `N {units} ago`   | N기간 이전의 전체 달력 단위. 예: **2주 전**, **3개월 전** |

## URL {#urls}

대시보드 URL에서 시간 쿼리를 조작할 수 있습니다.

다음 대시보드 URL을 참고하세요.

```
https://app.datadoghq.com/dash/host/<DASHBOARD_ID>?from_ts=<QUERY_START>&to_ts=<QUERY_END>&live=true
```

- `from_ts` 파라미터는 쿼리 시작 시간의 Unix 밀리초 타임스탬프입니다. 예를 들어, `1683518770980`입니다.
- `to_ts` 파라미터는 쿼리 종료 시간의 Unix 밀리초 타임스탬프입니다. 예를 들어, `1683605233205`입니다.
- `live=true` 은 쿼리를 저장하거나 공유할 때 상대 시간 지정이 유지됨을 나타냅니다. `live=false`도 사용할 수 있습니다.
