<div id="cdoc-content" class=""><article><h2 id="overview">Overview</h2><p>The Datadog .NET MAUI SDK for RUM generates events with associated telemetry and attributes. Telemetry are quantifiable values that can be used for measurements related to the event. Attributes are non-quantifiable values used to slice telemetry values (group by) in the RUM Explorer.</p><p>Most .NET MAUI Monitoring data is collected by the native Datadog iOS and Android SDKs for RUM, and is retained for the same periods of time.</p><ul><li>For iOS event-specific telemetry and attributes, see <a href="/real_user_monitoring/application_monitoring/ios/data_collected/#event-specific-attributes">RUM iOS Data Collected</a>.</li><li>For Android event-specific telemetry and attributes, see <a href="/real_user_monitoring/application_monitoring/android/data_collected/#event-specific-attributes">RUM Android Data Collected</a>.</li></ul><h2 id="what-s-collected-by-the-net-maui-layer">What's collected by the .NET MAUI layer</h2><p>These events originate from the .NET MAUI SDK before being forwarded to the native iOS or Android SDK. They use the same event schemas as iOS and Android above, with the following MAUI-specific notes:</p><table><thead><tr><th>Event</th><th>Trigger</th><th>Notes</th></tr></thead><tbody><tr><td><strong>View</strong></td><td>MAUI page navigation (<code>Application.PageAppearing</code>) — including Shell route changes, <code>Navigation.PushAsync</code>, and modals</td><td><code>view.name</code> is the resolved Shell route (for example, <code>MainPage/DetailPage</code>) or, for non-Shell navigations, the page class name. See <a href="/real_user_monitoring/application_monitoring/maui/advanced_configuration/#view-naming-priority">view naming priority</a>.</td></tr><tr><td><strong>Action</strong></td><td>Taps on <code>Button</code>, <code>ImageButton</code>, <code>Switch</code>, <code>CheckBox</code>, <code>Picker</code>, and <code>TapGestureRecognizer</code>/<code>SwipeGestureRecognizer</code></td><td><code>action.target.name</code> is resolved from <code>AutomationId</code> → <code>StyleId</code> (<code>x:Name</code>) → control type name. See <a href="/real_user_monitoring/application_monitoring/maui/advanced_configuration/#action-target-naming-priority">action target naming priority</a>.</td></tr><tr><td><strong>Resource</strong></td><td>Every <code>HttpClient</code> request, tracked via <code>DiagnosticListener</code> (includes requests issued by third-party libraries)</td><td>Captured at the .NET layer rather than the platform layer, so requests issued by managed code are visible even when they don't surface through the native HTTP stack.</td></tr><tr><td><strong>Error</strong></td><td>Unhandled C# exceptions (<code>AppDomain.UnhandledException</code>), unobserved task exceptions (<code>TaskScheduler.UnobservedTaskException</code>), and (optionally) native iOS/Android crashes when <code>NativeCrashReportEnabled = true</code></td><td>C# errors include the managed stack trace; native crashes include the platform stack trace. See <a href="/real_user_monitoring/application_monitoring/maui/error_tracking/">Crash Reporting and Error Tracking</a>.</td></tr></tbody></table><h2 id="disabling-automatic-data-collection">Disabling automatic data collection</h2><p>In Datadog, navigate to <a href="https://app.datadoghq.com/rum/list"><strong>Digital Experience</strong> &gt; <strong>Performance Monitoring</strong> &gt; <strong>Application List</strong></a> to access your application's settings. To stop collecting client IPs or geolocation data, uncheck the corresponding settings.</p><p>To customize or disable individual MAUI trackers in code, see <a href="/real_user_monitoring/application_monitoring/maui/advanced_configuration/#customize-automatic-tracking">Advanced Configuration &gt; Customize automatic tracking</a>.</p><h2
