--- title: DD-Agentの基本的な使用方法 (Windows) ---

概要

このガイドでは、Datadog Agentの基本的な機能を概説します。 まだDatadog Agent のインストール済ませていない場合は、Datadog にloginした状態で、該当OSの Agent ページのインストール手順も合わせてご参照ください。

Datadog Agent の起動と停止

Datadog Agentの実行は、Windows serviceによって管理されています。

Datadog Agent バージョン >= 3.9.1の場合:

スタートアップメニューに表示されるDatadog Agent Managerを使います。

Datadog Agent バージョン < 3.9.1の場合:

サービスパネルからDatadog Agentの起動, 停止, 再起動をします。サービスパネルの画面を表示するには、コマンドプロンプトcmd.exe shell にて次のコマンドを実行します。 <%= console 'services.msc' %> コンソールに入ったら、"Datadog Agent"というサービスを探します。名前にカーソルを合わせ、右クリックをすると起動, 停止, 再起動のオプションが表示されます。

動作ステータスの確認

Datadog Agent が動作していることを確認するには、サービスパネルのステータースが"Started"になっていることを確認します。 その際、"ddagent.exe" が、プロセスがタスクマネージャーに表示されていることを確認してください。 Datadog Agent の状況に関し更なる情報を取得するには、ブラウザーで次のURLのstatus pageを表示します。 <%= console 'http://localhost:17125/status' %> このステータス・ページは、Datadog Agent 3.9.1以降に実装されています。

設定ファイルの保存されているディレクトリ

Datadog Agent バージョン >= 3.9.1の場合:

スタートメニューにあるDatadog Agent Manager から、設定ファイルのチェック, 起動, 停止ができます。 設定ファイルの変更を適応するには、Datadog Agent の再起動が必要です。

Datadog Agent バージョン < 3.9.1の場合:

設定ファイルの保存場所は、Windowsのバージョンによって異なります。 Windows Server 2003, XP 又は、それ以前のバージョン:
  • Datadog Agent の設定ファイルの保存先:
    C:\Documents and Settings\All Users\Application Data\Datadog\datadog.conf
  • Integrations の設定ファイルの保存先:
    C:\Documents and Settings\All Users\Application Data\Datadog\conf.d\
Windows Server 2008, Vista と、それ以降のバージョン:
  • Datadog Agent の設定ファイルの保存先:
    C:\ProgramData\Datadog\datadog.conf
  • Integrations の設定ファイルの保存先:
    C:\ProgramData\Datadog\conf.d\

トラブルシューティング

最初にサービスパネルとタスクマネージャーでDatadog Agent 動作していることを確認してください。 次に、ステータース・ページをブラウザーで表示し、Datadgo Agent の状況を確認します。

Datadog Agent バージョン >= 3.9.1の場合:

ログの保存場所:
  • Windows Server 2003, XP 又は、それ以前のバージョン:
    C:\Documents and Settings\All Users\Application Data\Datadog\logs\ddagent.log
  • Windows Server 2008, Vista と、それ以降のバージョン:
    C:\ProgramData\datadog\logs\ddagent.log

Datadog Agent バージョン < 3.9.1の場合:

Datadog Agent の各要素のログは、”Event Viewer”で確認できます。Windows Logs → Application へとメニューを選択してください。
