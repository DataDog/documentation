--- title: DD-Agentの基本的な使用方法 (Debian系) ---

概要

このガイドでは、Datadog Agentの基本的な機能を概説します。 まだDatadog Agent のインストール済ませていない場合は、Datadog にloginした状態で、該当OSの Agent ページのインストール手順も合わせてご参照ください。

Datadog Agent の起動と停止

手動で Datadog Agent を起動する場合: <%= console 'sudo /etc/init.d/datadog-agent start' %> 停止する場合: <%= console 'sudo /etc/init.d/datadog-agent stop' %> リスタートする場合: <%= console 'sudo /etc/init.d/datadog-agent restart' %>

動作ステータスの確認

Datadog Agent が起動しているか確認する場合: (ver. 3.8.0 ~) <%= console 'sudo /etc/init.d/datadog-agent status' %> Datadog Agent のステータスに関する情報を収集する場合: <%= console 'sudo /etc/init.d/datadog-agent info' %> エラーをトレスバックする場合: -v フラッグをinfoコマンドに追記する。 (ver. 3.8.0 ~) <%= console 'sudo /etc/init.d/datadog-agent info -v' %>

設定ファイルの保存されているディレクトリ

Datadog Agent の設定ファイルの保存先 /etc/dd-agent/datadog.conf インストールしたIntegrations の設定ファイルの保存先 /etc/dd-agent/conf.d/

トラブルシューティング

info コマンドを実行することによって、Datadog Agentのステータス情報を表示することができます。 それぞれの細分化された要素(機能別)のログ関しては、以下のファイルに記録されています:
  • /var/log/datadog/supervisord.log (ver. 3.8.0 ~)
  • /var/log/datadog/collector.log
  • /var/log/datadog/dogstatsd.log
  • /var/log/datadog/forwarder.log

上記の方法を実行しても問題が解決しない場合は、 サポートチームにご連絡ください。
