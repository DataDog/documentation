## 目標
インタラクティブセッションで、EC2 インスタンスメタデータサービスへのアクセスにネットワークユーティリティ (`cURL` または `Wget` など) が使用されたことを検出します。

## 戦略
AWS EC2 IMDS は、IAM セキュリティ認証情報など、指定された EC2 インスタンスに関するデータを提供するリンクローカルの HTTP エンドポイントです。セキュリティ認証情報の取得に IMDS を使用できるため、攻撃者は他の AWS リソースにアクセスするために権限をエスカレートする場合があります。そこで、この検出機能により、Linux ネットワークユーティリティがインタラクティブセッションで IMDS へのアクセスに使用されたことを特定します。特に本番環境では、この動作がインタラクティブに発生することは不定期です。


## トリアージと対応
1. これが予期された行動であるか（たとえば、ある従業員が承認済みの理由でコマンドを実行したのか、コンフィギュレーション管理ユーティリティがインタラクティブセッションを使用したのか、など）を見極めます。
2. 動作が予期しないものである場合は、侵害の封じ込めを試み（攻撃の段階によっては、ワークロードを終了することで）、最初の侵害の兆候を探します。組織の内部プロセスに従い、侵害されたシステムを調査および修正します。
3. 攻撃の性質と関連するツールを特定します。イベントの前後に発生するセキュリティシグナル (存在する場合) を調査して、攻撃パスを確立します。
4. エクスプロイトの根本原因を見つけて修復します。
