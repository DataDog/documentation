## 目標
一般的なシェルユーティリティ、HTTP ユーティリティ、または Java プロセスによって生成されたシェルを検出します。

## 戦略
多くのアプリケーション (一部のデータベース、Web サーバー、検索エンジンなど) は Java プロセスとして実行されます。攻撃者は、これらのアプリケーションで構築されたプログラムの欠陥を利用する可能性があります (たとえば、Java プロセスとして実行されているデータベースへの SQL インジェクション)。この検出は、Java プロセスが一般的なシェルユーティリティ、HTTP ユーティリティ、またはシェルを生成したときにトリガーされます。これが予期しない動作である場合は、攻撃者がホストを危険にさらそうとしていることを示している可能性があります。

## トリアージと対応
1. Java プロセスの性質と目的を決定します。
2. Java プロセスがシェルとユーティリティを実行するための承認された目的があるかどうかを判断します。
3. この動作が予期しないものである場合は、侵害の封じ込めを試みます (攻撃の段階によっては、ワークロードを終了することで実現できる場合があります)。アプリケーションログまたは APM データを調査して、最初の侵害の兆候を探します。侵害されたシステムを調査および修正するために、組織の内部プロセスに従います。
4. エクスプロイトの根本原因を見つけて修復します。
