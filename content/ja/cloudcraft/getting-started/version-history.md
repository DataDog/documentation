## 概要

バージョン履歴は、アーキテクチャ ダイアグラムの変更を時系列で追跡し、過去のバージョンを確認して復元できる機能です。複雑なクラウド アーキテクチャを管理している場合やチームで共同作業をしている場合でも、バージョン履歴はダイアグラムの進化を把握するための貴重なインサイトを提供します。

Cloudcraft アプリケーションの右上にある **Version history** ボタンをクリックすると、バージョン履歴機能にアクセスできます。

## バージョンの操作

バージョン履歴は過去の作業を確認するだけでなく、現在および将来のダイアグラムを管理するための強力なツールでもあります。以下の操作が可能です。

1. **Restoring previous versions**: 以前の状態へ戻したい場合、過去バージョンを表示した状態で画面右上の **Restore this version** ボタンをクリックするだけで簡単に復元できます。
2. **Creating new blueprints**: 任意のバージョンを新しい Blueprint として保存できます。テンプレート作成や特定のアーキテクチャ状態を将来参照用に保存したい場合に便利です。バージョン名の右側にある 3 点アイコンをクリックし、**Save as a new blueprint** を選択してください。
3. **Comparing versions**: 明示的な比較機能はありませんが、異なるバージョンを表示することでアーキテクチャがどのように変化したかを手動で比較できます。

**注**: バージョンを手動で作成・削除する機能はありません。

### 作成

ダイアグラムを編集するとバージョンが自動的に作成されます。デフォルトでは作成日時がタイムスタンプとしてバージョン名に付与されますが、より分かりやすくするために特定のバージョンへカスタム名を設定できます。これはアーキテクチャの重要なマイルストーンや変更を示す際に特に有用です。

バージョンに名前を付ける手順

1. 名前を付けたいバージョンを選択します。
2. バージョン名の右側にある 3 点アイコンをクリックします。
3. ドロップダウン メニューから **Name this version** を選択します。

{{< img src="cloudcraft/getting-started/version-history/version-history-interface-cloudcraft.png" alt="Cloudcraft の Version history インターフェイスで、バージョンに名前を付けたり保存したりするオプションが表示されています。" responsive="true" style="width:100%;">}}

Cloudcraft は粒度と効率のバランスを取るため、賢く新しいバージョンを生成します。現在のバージョンから 5 分以上経過していれば、更新内容は新しいバージョンを作成します。最新バージョンから 5 分以内の変更はそのバージョンに追加されます。これにより履歴が細かい変更で煩雑になるのを防ぎつつ、有用性を保ちます。

手動でバージョンを作成することはできませんが、ダイアグラムを [Snapshot から Live mode へ切り替え][1]た際には新しいバージョンが自動的に生成される点に注意してください。

### メタデータ

各バージョンにはユーザー名やタイムスタンプなどのメタデータが含まれます。

バージョン名の右側にはバージョンを作成したユーザー名が表示され、別のユーザーが最後に編集した場合はそのユーザー名も表示されるため、変更の責任範囲が明確になります。

{{< img src="cloudcraft/getting-started/version-history/cloudcraft-version-history-user-timestamps.png" alt="ユーザーの詳細とタイムスタンプが表示されたバージョン履歴インターフェイス。" responsive="true" style="width:100%;">}}

Live ダイアグラムの場合、バージョン名の左に緑色の稲妻アイコンが表示され、Snapshot バージョンと区別できます。

### 検索

履歴を効率的にナビゲートできるよう、Cloudcraft には検索機能があります。バージョン名または日付で検索でき、ダイアグラムのタイムライン上の特定ポイントを簡単に見つけられます。

バージョンを検索するには、バージョン履歴パネル上部の検索バーにクエリを入力します。

大きな変更点に集中したい場合は、検索バー下の **Only show named versions** チェックボックスをオンにして無名のバージョンを非表示にし、履歴ビューを整理できます。

### 保持期間

名前付きのバージョンは無期限で保持され、重要なダイアグラム状態を恒久的に保存します。

無名のバージョンの保持期間はプランにより異なります。
- Free と Pro プラン: 30 日間保持
- Enterprise プラン: 90 日間保持

この階層型ポリシーにより、ライト ユーザーは十分な履歴を維持しつつ、より複雑なニーズを持つ Enterprise 顧客には拡張された保持期間を提供します。

## ベスト プラクティス

バージョン履歴機能を最大限活用するため、以下を推奨します。

1. **重要なバージョンに名前を付ける**: 意味のある名前を付けることで、アーキテクチャの重要な段階を容易に識別でき、永続的に保持されます。
2. **定期的にレビューする**: バージョン履歴を定期的に確認し、アーキテクチャの進化を追跡します。これは長期的な設計判断の洞察につながります。
3. **検索を活用する**: 検索機能と **Only show named versions** フィルターを組み合わせ、大量の Iteration があるプロジェクトでも迅速に目的のバージョンに到達できます。
4. **保持期間を計画する**: Free または Pro プランの場合、無名のバージョンの保持期間が 30 日である点を考慮し、それ以降も残したい場合は名前を付けてください。
5. **共同ドキュメントとして活用する**: チームで作業する場合、バージョン名に誰が何を変更したかを記録すると、チーム内のコミュニケーション ツールとして役立ちます。
6. **提案用にバージョンを利用する**: 大規模な変更を行う前に名前付きのバージョンを作成すると、提案が承認されなかった場合でも簡単にロールバックできます。
