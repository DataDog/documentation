title: recurso
---
1. En APM, un recurso es un dominio particular de una aplicación, por lo general un endpoint web instrumentado, una consulta de base de datos o un trabajo en segundo plano.
2. En RUM, un recurso es un tipo de evento. Se genera un evento de recurso para imágenes, bibliotecas JS, XHR, Fetch o CSS cargadas en una página.
3. En Cloud Security Management Misconfigurations, un recurso es una entidad configurable que se debe analizar de manera continua para comprobar que cumple con uno o más controles. Ejemplos de recursos de instancias de AWS incluyen hosts, contenedores, grupos de seguridad, usuarios y políticas de IAM gestionadas por el cliente.
