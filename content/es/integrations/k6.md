## Información general

[k6](https://k6.io/open-source) es una herramienta de tests de carga de código abierto que te ayuda a detectar problemas de rendimiento y regresiones de forma temprana.

Con la integración de k6, puedes realizar un rastreo de las métricas de rendimiento de tests de k6 para:

- Correlacionar el rendimiento de la aplicación con las métricas de tests de carga.
- Crear alertas basadas en métricas de tests de rendimiento.
- Analiza y visualiza métricas de k6 utilizando el dashboard de k6 Datadog o el [Explorador de métricas](https://docs.datadoghq.com/metrics/explorer/).

![Dashboard de k6 Datadog](https://raw.githubusercontent.com/DataDog/integrations-extras/master/k6/images/k6-default-dashboard.png)

## Configuración

Para obtener instrucciones detalladas, consulta la [documentación de k6](https://grafana.com/docs/k6/latest/results-output/real-time/datadog/).

### Instalación

1. En Datadog, ve a Integrations (Integraciones) > [API](https://app.datadoghq.com/organization-settings/api-keys) y copia tu clave de API.

1. Ejecuta el Datadog Agent:

   Para introducir métricas de k6 en Datadog, k6 envía métricas a través del Datadog Agent, que las recopila, agrega y reenvía a la plataforma Datadog.

   Ejecuta el servicio Datadog Agent como un contenedor Docker con este comando:

   ```shell
   DOCKER_CONTENT_TRUST=1 \
   docker run -d \
       --name datadog \
       -v /var/run/docker.sock:/var/run/docker.sock:ro \
       -v /proc/:/host/proc/:ro \
       -v /sys/fs/cgroup/:/host/sys/fs/cgroup:ro \
       -e DD_SITE="datadoghq.com" \
       -e DD_API_KEY=<YOUR_DATADOG_API_KEY> \
       -e DD_DOGSTATSD_NON_LOCAL_TRAFFIC=1 \
       -p 8125:8125/udp \
       datadog/agent:latest
   ```

   **Nota**: Sustituye `<YOUR_DATADOG_API_KEY>` por tu clave de [API](https://app.datadoghq.com/organization-settings/api-keys). Si tu cuenta está registrada con Datadog EU, cambia el valor de `DD_SITE` por `datadoghq.eu`.

1. Ejecuta el test de k6 y envía los resultados a Datadog.

   Una vez ejecutado el servicio Datadog Agent, ejecuta el test k6 y envía las métricas al Agent con:

   ```shell
   K6_STATSD_ENABLE_TAGS=true k6 run --out xk6-output-statsd script.js
   ```

1. Visualiza las métricas de k6 en Datadog.

   Mientras ejecuta el test, k6 envía métricas periódicamente a Datadog. Por defecto, estas métricas tienen `k6.` como prefijo del nombre.

   Puedes visualizar las métricas de k6 en tiempo real a través del [Explorador de métricas](https://docs.datadoghq.com/metrics/explorer/), los [monitores](https://docs.datadoghq.com/monitors/) o los [dashboards personalizados](https://docs.datadoghq.com/dashboards/).

   ![Explorador de métricas de k6 Datadog](https://raw.githubusercontent.com/DataDog/integrations-extras/master/k6/images/metrics-explorer.png)

   Además, la primera vez que Datadog detecta la métrica `k6.http_reqs`, el cuadro de la integración k6 se instala automáticamente y el dashboard k6 predeterminado se añade a tu [lista de dashboards](https://docs.datadoghq.com/dashboards/list).

   ![Dashboard de k6 Datadog](https://raw.githubusercontent.com/DataDog/integrations-extras/master/k6/images/k6-default-dashboard.png)
