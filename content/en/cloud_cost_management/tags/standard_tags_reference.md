This page is a detailed reference for every standard tag available on Cloud Cost Management data, organized by provider. For an overview of where tags come from, how they are prioritized, and how they are normalized, see [Tags in Cloud Cost Management][1]. For cost metric definitions, see [Metrics][2].

## FOCUS — The cross-provider standard

FOCUS (FinOps Open Cost and Usage Specification) tags are provider-agnostic tags added to **all** cost data — cloud providers, SaaS integrations, Datadog costs, and custom costs. They allow querying similar concepts across every cost source using a single tag name.

### FOCUS tag definitions (17 tags)

| Tag | Description |
|-----|-------------|
| `providername` | The cloud provider or cost source name (e.g., `aws`, `azure`, `gcp`, `oci`, `datadog`, `Snowflake`) |
| `publishername` | Entity making resources or services available for purchase |
| `servicename` | An offering that can be purchased from a provider |
| `billingaccountid` | Identifier assigned to a billing account by the provider |
| `billingaccountname` | Display name assigned to a billing account |
| `subaccountid` | ID assigned to a grouping of resources or services |
| `subaccountname` | Name assigned to a grouping of resources or services |
| `regionname` | Name of an isolated geographic area where a resource is provisioned |
| `regionid` | Provider-assigned identifier for a region |
| `availabilityzone` | Provider-assigned identifier for a physically separated area within a region |
| `pricingunit` | Provider-specified measurement unit for determining unit prices |
| `billingcurrency` | Currency in which a cloud bill was paid |
| `chargecategory` | Highest-level classification of a charge (see values below) |
| `chargefrequency` | How often a charge will occur (see values below) |
| `chargedescription` | Description of a charge |
| `skumeter` | Additional provider-specific SKU detail |
| `invoiceissuer` | Entity that issued the invoice |

## Custom costs tags

When uploading custom cost files, these FOCUS-aligned fields are required and become queryable tags:

| Field (CSV/JSON) | Tag (normalized) | Description |
|-------------------|-----------------|-------------|
| `ProviderName` | `provider_name` | The service being consumed (e.g., "Internal Platform") |
| `ChargeDescription` | `charge_description` | What aspect of a service is being charged |
| `ChargePeriodStart` | `charge_period_start` | Start day of a charge (YYYY-MM-DD) |
| `ChargePeriodEnd` | `charge_period_end` | Last day of a charge, inclusive (YYYY-MM-DD) |
| `BilledCost` | `billed_cost` | The amount being charged |
| `BillingCurrency` | `billing_currency` | Currency of billed cost (must be USD) |
| `UsageQuantity` | `usage_quantity` | Optional: usage amount |
| `UsageUnit` | `usage_unit` | Optional: usage unit |
| `Tags` | *(expanded)* | Optional: JSON object whose properties become individual tags |

Any additional CSV columns beyond these also become user-defined tags on the cost data. FOCUS tag mappings for custom costs are documented in [How each provider maps to FOCUS](#how-each-provider-maps-to-focus).
