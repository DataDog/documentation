[Workload Protection][1] security signals are created when Datadog detects a threat based on a security rule. View, search, filter, and investigate security signals in the [Signals Explorer][2], or configure [Notification Rules][3] to send signals to third-party tools.

To modify security signals, you must have the `security_monitoring_signals_write` permission. See [Role Based Access Control][4] for more information about Datadog's default roles and granular role-based access control permissions available for Cloud Security.

{{< img src="security/workload_protection/workload_protection_signal.jpg" alt="Workload Protection signal with an investigation graph and threat timeline" width="100%">}}

## Signals Explorer

The [Signals Explorer][2] lists Workload Protection security signals generated by [detection rules][5]. Use the facet panel or search bar to filter signals by severity, triage state, detection rule, host, container, or other attributes.

To filter signals by triage state, use the search query `@workflow.triage.state:<status>`, where `<status>` is the state you want to filter on (`open`, `under_review`, or `archived`). You can also use the **Signal State** facet on the facet panel.

Select a signal to open the side panel. From there, you can [investigate the threat][6] using the investigation graph, timeline, context, and Signal JSON, or [take action][7] to triage, escalate, automate, or remediate the signal.
