<div id="cdoc-content" class=""><article><div class="d-none site-region-container" data-region="gov,gov2"><div class="alert alert-danger"><p>RUM Auto-Instrumentation is not available for the selected site (<span class="cdoc js-region-param region-param" data-region-param="dd_site_name"></span>). Use <a href="/real_user_monitoring/application_monitoring/browser/setup/client">Client-Side instrumentation</a> instead.</p></div></div><h2 id="overview">Overview</h2><p>The Datadog Browser SDK enables Real User Monitoring (RUM) for your web applications, providing comprehensive visibility into user experience and application performance. With RUM, you can monitor page load times, user interactions, resource loading, and application errors in real-time.</p><p>There are three ways to instrument your browser-based web applications with the Browser SDK:</p><ul><li><a href="/real_user_monitoring/application_monitoring/browser/setup/client"><strong>Manual client-side setup</strong></a>: Instrument your browser-based web applications, deploy them, configure the initialization parameters you want to track, and use advanced configuration to further manage the data and context that RUM collects.</li><li><a href="/real_user_monitoring/application_monitoring/agentic_onboarding/?tab=realusermonitoring"><strong>Agentic Onboarding</strong></a>: Instrument your frontend applications with one prompt using LLM coding agents like Cursor or Claude.</li><li><a href="/real_user_monitoring/application_monitoring/browser/setup/server"><strong>Auto-Instrumentation</strong></a>: Inject a RUM SDK JavaScript scriptlet into the HTML responses of your web applications being served through a web server or proxy.</li></ul><h2 id="how-to-choose-the-instrumentation-type">How to choose the instrumentation type</h2><table><thead><tr><th></th><th><a href="/real_user_monitoring/application_monitoring/browser/setup/client">Manual client-side setup</a></th><th><a href="/real_user_monitoring/application_monitoring/agentic_onboarding/?tab=realusermonitoring">Agentic Onboarding (Preview)</a></th><th><a href="/real_user_monitoring/application_monitoring/browser/setup/server">Auto-Instrumentation (Preview)</a></th></tr></thead><tbody><tr><td><strong>SDK setup mechanism</strong></td><td>Manually add the RUM SDK to your application code and manage configurations in code.</td><td>Use an AI-guided setup that detects your project's framework and adds the RUM SDK with a single prompt.</td><td>Automatically add RUM JS to your web app HTML, then manage configurations in the UI.</td></tr><tr><td><strong>Code changes required</strong></td><td>Yes</td><td>Yes (automated by AI agent)</td><td>No</td></tr><tr><td><strong>Setup complexity</strong></td><td>Medium</td><td>Low</td><td>Low</td></tr><tr><td><strong>Supported platforms</strong></td><td>All browser-based applications</td><td>Next.js, React, Svelte, Vue, Vanilla JavaScript</td><td>Apache, IBM HTTP Server, Java Servlet, Nginx, Windows IIS</td></tr><tr><td><strong>User groups</strong></td><td>Frontend, mobile engineering, or product teams who work directly in application code</td><td>Teams using AI coding agents such as Cursor or Claude Code</td><td>SRE and engineering teams without frontend access who need centralized management</td></tr></tbody></table><p>This page describes how to instrument your web applications with the Datadog Browser SDK. The Browser SDK supports <a href="/real_user_monitoring/application_monitoring/browser/setup/client">Real User Monitoring (RUM)</a>, <a href="/error_tracking/frontend/browser">Error Tracking</a>, <a href="/session_replay/browser/">Session Replay</a>, and <a href="/product_analytics/">Product Analytics</a>.</p><p>Select a setup method based on your application stack and workflow:</p><ul><li><strong>Manual client-side setup</strong>: Add the SDK directly to your frontend code when you need full control over initialization and configuration.</li><li><strong>Agentic Onboarding</strong>: Use an AI-assisted workflow to automatically configure and deploy the SDK with minimal manual steps.</li><li><strong>Server-side auto-instrumentation (RUM only)</strong>: Inject the SDK through your web server or proxy when you can't modify frontend code directly.</li></ul><p>The Browser SDK supports all modern desktop and mobile browsers.</p><h2 id="setup">Setup</h2><details
