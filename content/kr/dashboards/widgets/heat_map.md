히트 맵 시각화는 *호스트*와 같은 여러 태그에서 집계된 메트릭들을 보여줍니다. 특정 값을 가진 호스트가 많을수록 사각형이 더 어두워집니다.

이 시각화는 단일 메트릭 쿼리만 표시하며 추가 쿼리들은 무시됩니다.

**참고**: 이 시각화에서는 이상값 감지를 수행할 수 없습니다.

## 구성

{{< img src="dashboards/widgets/heat_map/heat_map_setup.png" alt="히트 맵 구성" style="width:80%;">}}

### 설정

평소와 같이 메트릭 쿼리를 구성하세요. **참고**: 이 시각화 유형은 메트릭들이 태그 키 전체에서 집계되는 경우(예: 각 `host`에 대해 집계되는 경우)에만 유용합니다.

"`avg`/`max`/`min`/`sum by`/etc."에서부터 선택하여 연관된 태그에서 데이터를 볼 수 있도록 컨트롤하세요.

### 옵션

#### 이벤트 오버레이

관련 시스템들로부터 이벤트들을 추가하여 그래프에 더 많은 컨텍스트를 추가하세요. 예를 들어 깃허브 커밋, 스테이징 환경의 이벤트 또는 접두사와 일치하는 컨테이너의 이벤트들을 추가할 수 있습니다. **Event Overlays** 섹션을 펼치고 쿼리를 입력하여 해당 이벤트들을 표시하세요. [Event Explorer][1]와 같은 쿼리 형식을 사용하세요. 아래 표의 예시들을 참조하세요.

| 질문                     | 설명                                                      |
|---------------------------|------------------------------------------------------------------|
| `source:(github OR chef)` | 깃허브 또는 Chef의 이벤트들을 표시합니다.                                 |
| `env:staging`             | `env:staging` 태그가 있는 이벤트들을 표시합니다.                          |
| `container_id:foo*`       | ID가 `foo`로 시작하는 모든 컨테이너의 이벤트들을 표시합니다. |

#### Y축 컨트롤

Y축 컨트롤들은 UI 및 JSON 편집기를 통해 사용할 수 있습니다.

축 컨트롤을 통해 다음을 수행할 수 있습니다:

* y축을 특정 범위로 자릅니다.
* 백분율 또는 절댓값 기준치에 따라 y축 범위를 자동으로 변경하세요. 이 임계값은 "아웃라이어" 시리즈를 제거하기 위해 그래프의 양쪽 끝(하한 및 상한) 중 하나에 적용할 수 있습니다. 
* y축 스케일을 선형에서 log, pow 또는 sqrt로 변경합니다.

*Y-Axis Controls* 버튼을 확장하여 Y축 스케일을 변경하세요.

다음 구성 옵션을 사용할 수 있습니다.

| 옵션                | 필수 | 설명                                                                                                                                                                                                       |
|-----------------------|----------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `Min`/`Max`           | 아니요       | y축에 표시할 최솟값 및/또는 최댓값을 지정하세요. 이것은 숫자 또는 `Auto`를 기본값으로 사용합니다.                                                                                                   |
| '스케일'               | 아니요       | 스케일 유형을 지정합니다. 가능한 값: <br>-*linear*: 선형 스케일(기본값) <br>-*log*: 로그 스케일 <br>-*pow*: 2의 거듭제곱 스케일(2는 기본값, JSON에서 수정) <br>-*sqrt*: 제곱근 척도 |
| `항상 0 포함` | 아니요       | 항상 0을 포함하거나 축을 데이터 범위에 맞추세요. 기본값은 항상 0을 포함하는 것입니다.                                                                                                                     |

**참고**: 수학적 로그 함수는 음수 값을 허용하지 않기 때문에 Datadog 로그 스케일은 값이 동일한 부호(모두 > 0 또는 모두 < 0)인 경우에만 작동합니다. 그렇지 않으면 빈 그래프가 반환됩니다.
