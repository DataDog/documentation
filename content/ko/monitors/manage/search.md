모니터를 검색하려면 왼쪽에 있는 패싯 패널이나 상단의 검색창을 사용해 쿼리를 작성합니다. 속성을 선택한 경우 검색창이 해당 쿼리로 업데이트됩니다. 마찬가지로 검색창 쿼리를 수정하거나 새 쿼리를 작성하면 속성 확인란이 업데이트되어 변경 사항이 반영됩니다. 어느 경우에나 쿼리를 편집하면 실시간으로 쿼리 결과가 업데이트됩니다. **참고**: 클릭할 *검색* 버튼이 없습니다.

## 검색창

모든 모니터 이름과 알림 메시지에 대해 단순한 텍스트를 사용해 검색하세요. 예를 들어 `*postgresql*`을 검색하면 이름이나 알림 메시지에 `postgresql`이 포함된 모든 모니터가 반환됩니다.

검색을 제한하려면 필드 이름을 지정하세요.

| 항목    | 설명                                            | 예시        |
|---------|--------------------------------------------------------|----------------|
| 타이틀   | 모니터 이름에서 텍스트로 검색하세요.                | `title:text`   |
| 메시지 | 모니터의 알림 메시지에서 텍스트로 검색하세요. | `message:text` |

또한, ID를 사용하여 검색할 수 있습니다. 예를 들어 `1234567`을 사용할 수 있습니다. 모니터 ID는 [모니터 상태 페이지][1]에 있습니다.

### 질문

부울 연산자(`AND`, `OR`, `NOT`)와 괄호를 사용해 검색 쿼리를 향상하세요. 검색 구문은 [Elasticsearch][2]와 유사하며 다음 예외가 적용됩니다.

* 정규식은 지원되지 않습니다.
* 단일 문자의 와일드카드(`?`)는 지원되지 않지만 일반적인 와일드카드(`*`)는 지원됩니다.
* 근접 검색은 지원되지 않지만 [퍼지][3] 연산자는 지원됩니다.
* 범위는 지원되지 않습니다.
* 부스팅(Boosting)은 지원되지 않습니다.

유지되는 문자: `-`, `(`, `)`, `"`, `~`, `*`, `:`, `.`, and whitespace. To search a monitor field that includes a reserved character, wrap the field string in quotes. For example, `status:Alert AND "chef-client"`는 올바른 쿼리 문자열입니다.

따옴표 붙은 필드와 관련해 몇몇 경고가 있습니다.

* 여러 필드에서 일반적으로 사용되므로 따옴표와 함께 또는 따옴표 없이 `.`을 쓸 수 있습니다. 예를 들어 `metric:system.cpu.idle`은 올바릅니다.
* 와일드카드 검색은 따옴표 붙은 문자열 내에서 지원되지 않습니다. 예를 들어 `"chef-client*"`에서 `*`이 문자 그대로 처리되므로 `"chef-client failing"`이란 이름의 모니터가 반환되지 않습니다.

## 속성

고급 검색을 통해 모든 모니터 속상 조합별로 모니터를 필터링할 수 있습니다.

| 속성    | 설명                                                                                     |
|--------------|-------------------------------------------------------------------------------------------------|
| 상태       | 모니터 상태: `Triggered` (`Alert`, `Warn`, `No Data`) 또는 `OK`                            |
| 음소거됨        | 모니터 음소거 상태: `true` 또는 `false`                                               |
| 유형         | Datadog [모니터 유형][4]                                                                   |
| 생성자      | 모니터 생성자                                                                      |
| 서비스      | 양식 `service:<VALUE>`에서 사용자가 사용한 서비스 태그입니다.                                         |
| 태그          | 모니터에 할당된 [태그](#monitor-tags)                                               |
| Env          | 양식 `env:<VALUE>`에서 사용자가 사용한 환경 태그입니다.                                         |
| 범위        | 검색 태그가 모니터 쿼리의 `from`필드에 나와 있습니다.                                   |
| 메트릭/점검 | 모니터링되는 메트릭 또는 서비스                                                      |
| 알림 | 알림을 받는 사람 또는 서비스                                                  |
| 왼쪽 음소거됨   | 다운타임으로 이 모니터에 대한 알림을 음소거하기 전까지 남은 시간입니다. `muted_left:30m`을 검색하면 최대 30분 동안 여전히 음소거 상태인 모든 모니터가 반환됩니다. 지원되는 단위는 초(`s`), 분(`m`), 시간(`h`) 및 주(`w`)입니다.    |
| 음소거 경과 시간 | 다운타임으로 이 모니터에 대한 알림이 음소거되기 시작한 후 경과한 시간입니다. `muted_elapsed:30d`을 검색하면 최소 30일 동안 음소거 상태인 모든 모니터가 반환됩니다. 지원되는 단위는 초(`s`), 분(`m`), 시간(`h`) 및 주(`w`)입니다. |

모니터를 찾기 위한 상자 수를 점검하세요. 다음 규칙이 적용됩니다.

* `AND` 연산자는 각기 다른 필드에서 속성을 점검할 때 적용할 수 있습니다. 예: `status:Alert type:Metric`(두 검색어 간 연산자 부족, `AND` 필요)
* 대부분의 경우 `OR` 연산자는 동일한 필드 내 속성을 점검할 때 적용됩니다. 예: `status:(Alert OR Warn)`. 일부 예외가 적용됩니다. 예를 들어 여러 범위나 서비스 태그를 점검할 때에는 `AND` 연산자가 사용됩니다.
* 일부 속성은 여러 값을 선택하도록 허용하지 않습니다. 예를 들어 메트릭이나 서비스 점검을 선택하면 선택 항목을 선택 해제할 때까지 목록에서 다른 옵션이 사라집니다.
* *상태* 속성 아래 있는 `Triggered` 확인란에 `status:(Alert OR Warn OR "No Data")`가 됩니다. 트리거됨은 올바른 모니터 상태가 아닙니다. 
* *메트릭/점검* 속성의 이름은 쿼리에서 항상 `metric`입니다. 예를 들어 점검 `http.can_connect`를 선택하면 `metric:http.can_connect`가 됩니다.

**참고**: 모니터 전반에서 많은 수의 속성 값이 있는 경우, 속성 검색창을 사용하여 올바른 값을 사용합니다.

### 기본 보기

{{< img src="monitors/manage_monitor/default.jpg" alt="기본 보기" style="width:50%;" >}}

기존 모니터 관리 보기가 저장된 기본 보기입니다. 이 설정은 사용자만 액세스하고 볼 수 있습니다. 설정을 업데이트하여도 조직에 영향을 미치지 않습니다.

UI에서 작업을 완료하거나 다른 설정을 포함하는 모니터 관리 페이지로의 링크를 여는 경우 **임시로** 기본 저장된 보기를 덮어쓸 수 있습니다.

기본 보기에서 보기 패널의 항목:

* 항목을 클릭하여 기본 보기를 **다시 로드**합니다.
* 현재 파라미터를 사용해 기본 보기를 **업데이트**합니다.
* 완전한 재시작을 위해 기본 보기를 Datadog 기본값으로 **재설정**합니다.
