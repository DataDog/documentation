## 개요

오른쪽 상단의 드롭다운 메뉴에서 시간 프레임을 선택한 후, [신서틱 모니터링 및 연속 테스트 탐색기][1]에서 **테스트 실행** 이벤트 유형을 클릭하여 테스트 실행을 검색할 수 있습니다.

패싯을 사용하여 다음 작업을 수행할 수 있습니다.

- 재시도가 필요한 최신 테스트 실행을 확인합니다.
- HTTP 상태 코드별로 실패한 API 테스트 실행을 집계하고 트렌드를 표시합니다. 

## 패싯 탐색하기

왼쪽 패싯 패널에 테스트 실행 검색에 사용할 수 있는 여러 패싯이 표시됩니다. 검색 쿼리를 사용자 지정하려면 **일반(Common)**으로 시작하는 패싯 목록을 클릭하세요.

### 일반 테스트 실행 속성

**일반(Common)** 패싯을 사용하면 테스트 실행 속성을 필터링할 수 있습니다.

| 패싯            | 설명                                                                                             |
|------------------|---------------------------------------------------------------------------------------------------------|
| `Batch ID`        | 테스트 실행과 연관된 브랜치 ID입니다.                                               |
| <code>실행&nbsp;규칙</code> | CI 배치의 테스트 결과와 연관된 실행 규칙입니다: `Blocking`, `Non Blocking`, `Skipped`. |
| `Location`       | 배치의 테스트 결과와 연관된 위치입니다.                                              |
| `Passed`        | 테스트 실행의 전체 상태입니다.                                               |
| `Run Type`      | 테스트 실행의 실행 유형입니다. CI, 수동 또는 예약하여 트리거할 수 있습니다.                                             |

### 시간 속성

**시간(Timings)** 패싯을 사용하면 API 테스트 실행에 대한 시간 관련 속성을 필터링할 수 있습니다.

| 패싯          | 설명                                 |
|----------------|---------------------------------------------|
| `DNS`  | API 테스트를 실행하기 위해 DNS 이름을 확인하는 데 소요된 시간입니다.  |
| `Download`     | API 테스트 응답을 다운로드하는 데 소요된 시간입니다.     |
| `First Byte`      | API 테스트를 실행하기 위해 응답의 퍼스트 바이트를 수신하는 데 걸린 시간입니다.      |
| `Open`  | WebSocket 테스트 실행을 위해  WebSocket이 오픈 상태로 유지된 전체 시간입니다.  |
| `Received` | WebSocket 연결이 WebSocket 테스트 실행을 위해 데이터를 수신하는 데 소요된 전체 시간입니다. |
| `TCP` | API 테스트 실행을 위해 TCP 연결을 설정하는 데 소요된 시간입니다. |
| `Total` | API 테스트 실행의 총 응답 시간입니다. |

### HTTP 속성

**HTTP** 패싯을 사용하면 HTTP 속성을 필터링할 수 있습니다.

| 패싯          | 설명                                 |
|----------------|---------------------------------------------|
| `HTTP Status Code`  | 테스트 실행을 위한 HTTP 상태 코드입니다.  |

### gRPC 속성

**gRPC** 패싯은 gRPC 테스트 실행과 연관되어 있습니다.

| 패싯       | 설명                               |
|-------------|-------------------------------------------|
| `Health Check Status`       | gRPC 테스트에 대한 건전성 점검 상태입니다. 상태는 `Serving` 또는 `Failing`로 표시됩니다.    |

### SSL 속성

**SSL** 패싯은 SSL 테스트 실행과 연관되어 있습니다.

| 패싯       | 설명                               |
|-------------|-------------------------------------------|
| `AltNames`       |SSL 인증서와 연관된 대체 레코드 이름입니다.    |

### TCP 속성

**TCP** 패싯은 테스트 실행 중 TCP 연결과 연관되어 있습니다.

| 패싯       | 설명                               |
|-------------|-------------------------------------------|
| `Connection Outcome`       | TCP 연결의 연결 상태입니다. 결과는 `established`, `timeout` 또는 `refused`로 표시됩니다.    |

재시도가 있었던 테스트 실행을 필터링하려면 `@result.isFastRetry:true`을 사용하여 검색 쿼리를 생성합니다. `@result.isLastRetry:true` 필드를 사용하여 재시도가 있었던 테스트의 마지막 실행을 검색할 수도 있습니다.

테스트 실행에 대한 더 자세한 정보는 [검색 신택스][2]를 참조하시기 바랍니다.
