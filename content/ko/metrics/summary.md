## 개요

[메트릭 요약 페이지][1]에는 지정된 기간(지난 시간, 지난일 또는 지난주) 동안 Datadog에 보고된 메트릭 목록을 표시됩니다.

**메트릭** 또는 **태그** 검색 필드를 사용해 메트릭 이름이나 태그별로 메트릭을 검색합니다.

{{< img src="metrics/summary/tag_advancedfiltering.mp4" alt="태그 검색 창에 NOT team:*이 입력된 메트릭 요약 페이지" video=true style="width:75%;">}}

부울 연산과 와일드카드 구문을 지원하는 태그 필터링으로 다음을 빠르게 파악할 수 있습니다.
* 특정 태그 키로 태그 된 메트릭(예: `team`:`team:*`)
* 특정 태그 키가 누락된 메트릭(예: `team`:`NOT team:*`)


## 패싯 패널

검색 창은 메트릭 목록을 필터링할 수 있는 가장 종합적인 작업 모음을 제공합니다. 하지만 패싯은 다음을 기준으로 빠르게 메트릭을 필터링할 수 있습니다.
* **설정**: 태그 설정 또는 추가 백분위수 집계를 통해 빠르게 메트릭을 파악하세요.
* **메트릭 유형**: 배포와 비배포(개수, 게이지, 속도)를 비교해 빠르게 파악하세요.
* **배포 메트릭 출처**: 배포 메트릭의 출처를 빠르게 파악하세요(예: 로그의 메트릭 생성, 스팬(span)의 메트릭 생성 등).

{{< img src="metrics/summary/facets2.jpg" alt="메트릭 패싯 패널" style="width:75%;">}}

## 메트릭 상세 정보 사이드 패널

메트릭 메타 데이터 및 태그와 관련된 자세한 정보를 보려면 아무 메트릭 이름을 클릭하여 상세 정보 사이드 패널을 표시하세요.

{{< img src="metrics/summary/mwl_sidepanel.jpg" alt="메트릭 패널" style="width:75%;">}}

### 메트릭 이름

[메트릭 탐색기][2], [대시보드][3] 등의 메트릭 이름입니다.

### 수집된 커스텀 메트릭

메트릭 이름은 연결된 태그 값 조합에 따라 수집된 여러 커스텀 메트릭을 내보낼 수 있습니다. 수집된 커스텀 메트릭은 원래 코드를 사용해 제출된 모든 데이터를 표시합니다.

[커스텀 메트릭][4] 설명서에서 자세히 알아보세요.

### 인덱스된 커스텀 메트릭

수집된 커스텀 메트릭과 달리 인덱스된 커스텀 메트릭은 Datadog 플랫폼 전반에서 쿼리할 수 있습니다. 이 수는 백분위수 집계를 추가하거나 제거하는 경우, 제한없는 메트릭 수집TM을 사용하는 경우 영향을 받을 수 있습니다. [제한없는 메트릭 수집TM][10] 설명서에서 자세히 알아보세요.

### 호스트

메트릭을 보고하는 총 호스트 개수입니다.

### 태그 값

메트릭에 연결된 고유한 태그 값의 총 수입니다.

[태깅에 대해 자세히 알아보세요][5].

### 메트릭 메타데이터

메트릭에 연결된 메타데이터입니다. 대부분의 메타데이터는 메트릭 요약 페이지에서 또는 [Datadog API][6]를 사용해 편집할 수 있습니다.

#### 메트릭 단위

메트릭 단위(바이트, 초, 요청, 쿼리 등)입니다. 자세한 정보는 [메트릭 단위][7] 페이지를 참조하세요.

커스텀 메트릭을 Datadog에 제출할 때 그래프의 메트릭 위를 마우스로 가리켜 [측정 단위][1]를 바꾸는 것이 가능합니다.

**참고**: 메트릭 그래프 표시 방법이 변경되지는 않습니다. 메트릭 위를 마우스로 가리키면 해당 원래 값의 측정 단위만 바뀝니다. 서식은 가독성을 위해 자동 적용됩니다. 예를 들어 바이트(`B`)는 킬로바이트(`KiB`)로 표시될 수 있습니다.

#### 메트릭 유형

메트릭 유형(게이지, 속도, 개수, 배포)입니다. 자세한 정보는 [메트릭 유형][8]을 참조하세요. 

**경고**: 메트릭 유형을 편집하면 **모든** 대시보드와 모니터에 대한 메트릭 행동이 변경됩니다.

#### 통합 이름

메트릭이 지원되는 [통합][9]에서 온 경우, 메타데이터는 통합 이름을 나열합니다. 이 정보는 편집할 수 없습니다.

#### 간격

메트릭의 수집 간격은 초 단위입니다.

#### 메트릭 설명

메트릭 설명은 메트릭 행동을 이해할 수 있도록 해줍니다. 설명은 지원되는 [통합][9] 메트릭에 대해 미리 채워져 있습니다. 이 필드를 사용하여 [커스텀 메트릭][4]에 대한 설명을 업데이트합니다.

### 태그 표

태그 표는 메트릭 데이터에서 활성 보고하는 모든 태그 키와 값을 살펴볼 수 있는 다양한 방법을 제공합니다.

테그 표를 사용하여 다음을 수행하세요.

- **열 개수**로 태크 키를 정렬합니다(고유한 태그 값 개수).
- 특정 태그 키에 대해 페이지 매긴 태그 표를 검색합니다.
- 태그 표를 다운로드 가능한 CSV 파일로 내보냅니다.
- 메트릭에서 제출한 태그와 자체적으로 메트릭에서 설정한 태그 간 토글합니다.

특정 키에 대해 다음을 수행하세요.

- 해당 태그 키의 모든 태그 값을 검사합니다.
- 특정 태그 `key:value`을(를) 사용해 메트릭 요약 페이지에 표시된 메트릭 목록을 상세하게 필터링합니다.
- 메트릭 탐색기에서 태그 `key:value` 페어별로 필터링한 메트릭 그래프를 엽니다.
- 적용 태그 필터링을 위해 `key:value` 테그를 복사합니다.

{{< img src="metrics/summary/updated_tags_table.mp4" alt="태그 표" video=true style="width:75%;">}}

[태깅에 대해 자세히 알아보세요][5].

## 제한 없는 메트릭 수집TM
제한없는 메트릭 수집TM은 에이전트나 코드 수준 변경 없이 커스텀 메트릭의 크기를 제어할 수 있도록 해줍니다.

**참고:** 제한없는 메트릭 수집TM은 커스텀 메트릭에 대해서만 사용할 수 있습니다.

메트릭 상세 정보 사이드 패널의 **태그 관리** 버튼이나 대량 메트릭 태그 설정 버튼을 사용해 태그를 구성할 수 있습니다.

{{< img src="metrics/distributions/managetags.png" alt="배포에서 태그 설정" style="width:80%;">}}

1. **메트릭 요약** 표에서 커스텀 배포 메트릭 이름을 클릭하여 메트릭 상세 정보 사이드 패널을 엽니다.
2. **태그 관리** 버튼을 클릭하여 태그 설정 모델을 엽니다.
3. **커스텀...** 탭을 클릭하여 쿼리를 위해 유지하려는 태그를 커스터마이즈합니다. 태그 설정은 유지하려는 태그의 _허용 목록_입니다.
4. 카디날리티 추측 도구를 사용해 제안된 허용 목록의 영향을 사전 검토한 다음 **저장**을 선택합니다.

**참고**: 태그 제외는 허용목록 기반 태그 커스터마이즈에서 지원되지 않습니다. `!`(으)로 시작하는 태그 추가는 허용되지 않습니다. 또한, 카디날리티 추측 도구는 48시간 보다 오래된 메트릭을 필요로 합니다.

### 쿼리 가능한 태그

제한없는 메트릭 수집TM을 통해 메트릭을 설정하면 쿼리 가능한 태그를 확인할 수 있습니다. 궁극적으로 이 볼륨이 _인덱스된 커스텀 메트릭 볼륨_에 기여합니다. _수집된 커스텀 메트릭_ 볼륨에 기여하는 원래 제출되고 수집된 태그로 다시 토글할 수 있습니다.

### 어드밴스드 모드의 집계를 활용해 메트릭 최적화

개수, 게이지, 속도 메트릭 유형의 커스텀 메트릭의 경우, 제한없는 메트릭 수집TM의 어드밴스드 모드의 추가 집계 등을 사용해 부수적으로 메트릭 설정을 상세 검색할 수 있습니다. 기본적으로 Datadog는 메트릭 유형에 따라 가장 자주 쿼리된 집계 조합을 저장하여 아래 나온 것처럼 설정된 메트릭 쿼리의 수학적 정확도를 유지합니다.

- 설정된 개수/속도는 `SUM` 시간/공간 집계를 사용해 쿼리할 수 있습니다.
- 설정된 게이지는 `AVG` 시간/공간 집계에서 쿼리할 수 있습니다.

{{< img src="metrics/summary/customize_aggr_docs.jpg" alt="개수, 속도 및 게이지 집계 상세 검색" style="width:80%;">}}

더 많은 집계도 사용할 수 있습니다. 에이전트 또는 코드 수준 변경 없이도 언제든지 집계를 추가하거나 제거할 수 있습니다.

**참고**: 개수, 속도 또는 게이지 메트릭을 설정하고 집계를 제거하면 기존 대시보드와 모니터에 영향을 미칠 수 있습니다.
