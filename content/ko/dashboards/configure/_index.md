## 개요

대시보드는 Datadog 제품 전반의 데이터에 대한 가시성을 제공합니다. 문제를 빠르게 해결할 수 있도록 세부 정보와 구성을 추가하고 시각화된 정보에 초점을 맞추세요.

개별 대시보드를 구성하여 다음과 같이 활용할 수 있습니다.
- [대시보드가 시각화하는 내용에 대한 세부 정보와 설명을 추가합니다](#dashboard-details)
- [맞춤형 대시보드 디스플레이를 설정하고, 대시보드 관련 활동을 확인하기 위해 구성 작업을 확인합니다](#configuration-actions)
- [권한을 사용하여 개별 대시보드에 대한 액세스를 제한합니다](#permissions)
- [템플릿 변수를 사용하여 맞춤형 뷰를 설정합니다](#template-variables)

## 대시보드 상세 정보

개별 대시보드에서 대시보드 제목 위에 마우스를 올려 놓으면 대시보드 세부 정보를 확인하고 편집할 수 있습니다. 제목과 작성자가 표시된 패널이 열립니다.

{{< img src="dashboards/suggested_dashboards.png" alt="편집 가능한 제목, 추천 대시보드 및 Teams 기능을 강조하는 대시보드 세부 정보" style="width:70%;" >}}

Markdown 지원 대시보드 설명을 업데이트하거나 [Teams][1]를 대시보드와 연동합니다.

1. 대시보드 제목 위를 마우스로 가리킵니다. 드롭다운 패널이 열립니다.
1. 대시보드 제목이나 설명을 클릭해 편집합니다.
1. 제목을 변경하려면 체크 버튼을 클릭합니다.
1. **팀** 드롭다운에서 최대 5개의 팀을 선택합니다.
1. (선택 사항) `[[suggested_dashboards]]`를 대시보드 설명 내에 추가하여 추천 대시보드 목록을 표시합니다. 이 대시보드들은 조직 내 사용자 활동과 사용자가 이 대시보드에서 다른 대시보드로 이동하는 빈도를 기반으로 추천됩니다.

## 템플릿 변수

 템플릿 변수를 사용하면 태그 또는 패싯을 기반으로 특정 호스트, 컨테이너, 서비스 하위 집합에 집중할 수 있도록 대시보드를 구성할 수 있습니다. 다음에 대한 자세한 내용은 [템플릿 변수][2] 문서를 참고하세요.
 - 대시보드 템플릿 변수 추가 및 구성
 - 대시보드 위젯에 템플릿 변수 적용
 - 템플릿 변수를 사용하여 Saved 뷰 생성

## 구성 작업

**Configure**를 클릭하면 대시보드에서 사용할 수 있는 구성 옵션 메뉴가 열립니다. 여기에는 다음이 포함됩니다.

| 구성    | 설명 |
| ----------- | ----------- |
| 비전 기록 | 대시보드 버전 히스토리를 미리 보거나 복원하거나 복제할 수 있습니다. 자세한 내용은 [Version History 가이드][3]를 참고하세요. |
| 감사 이벤트 보기 | 조직 내에서 이 대시보드를 사용하는 사람을 확인하세요. 개인 사용자는 자신의 작업 내역을 확인할 수 있습니다. 자세한 내용은 [Datadog Audit Trail][4]을 참고하세요. |
| 대시보드 복제 | 전체 대시보드를 새 대시보드로 복사합니다. 복제본의 이름을 지정하라는 메시지가 표시됩니다. |
| 키보드 단축키 | 사용 가능한 단축키 목록을 확인합니다. |
| UTC 시간 표시 | UTC 시간과 기본 시간대 사이를 전환합니다. |
| 밀도 증가 | 고밀도 모드는 대시보드에 그룹 위젯을 나란히 표시하여 위젯 밀도를 높입니다. 그룹 위젯을 사용하는 대시보드의 대형 화면에서는 이 모드가 기본적으로 켜집니다. |
| TV 모드 | 주요 성능 메트릭을 대형 화면이나 TV에 표시하도록 전환합니다. 자세한 내용은 [대시보드에 TV 모드 사용][5]을 참고하세요. |

### 알림

대시보드 변경 알림을 받으려면 알림 추적을 활성화하세요. 조직의 모든 사용자는 관리자 권한에 관계없이 이 기능을 직접 활성화할 수 있습니다.

대시보드  알림이 활성화되면 [Events Explorer][6]에 이벤트가 생성됩니다. 이 이벤트는 텍스트 변경, 위젯 변경, 대시보드 복제, 대시보드 삭제에 관한 정보와 함께 작업한 사용자의 이름을 표시합니다. Events Explorer에서 특정 대시보드에 대한 변경 이벤트를 보려면 다음을 검색하세요.

```text
tags:(audit AND dash) <DASHBOARD_NAME>
```

### 대시보드 JSON 복사, 가져오기 또는 내보내기

다음 옵션과 함께 내보내기 아이콘(오른쪽 상단)을 사용하여 대시보드의 JSON을 복사, 가져오기, 내보내기합니다.

| 옵션                          | 설명                                                                                                                                                                |
|---------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Copy&nbsp;dashboard&nbsp;JSON   | 대시보드의 JSON을 클립보드에 복사합니다.                                                                                                                               |
| Import&nbsp;dashboard&nbsp;JSON | JSON을 대시보드에 붙여넣거나 가져옵니다. 이 옵션은 대시보드의 모든 콘텐츠를 덮어씁니다. JSON이 이미 클립보드에 있는 경우 `Ctrl V`(Mac의 경우 `Cmd V`)을(를) 사용합니다. |
| Export&nbsp;dashboard&nbsp;JSON | 대시보드의 JSON이 포함된 JSON 파일을 다운로드합니다.                                                                                                                |

### 대시보드 삭제

<div class="alert alert-warning">대시보드는 삭제 전 별표를 해제해야 합니다.</div>

이 옵션을 사용하면 대시보드를 영구적으로 삭제할 수 있습니다. 미리 설정된 **Recently Deleted** 목록을 사용하여 삭제된 대시보드를 복원할 수 있습니다. **Recently Deleted** 목록에 있는 대시보드는 30일 후에 영구적으로 삭제됩니다. 자세한 내용은 [대시보드 목록][7] 문서를 참고하세요.

## 권한

<div class="alert alert-info">유료 플랜을 사용하는 모든 사용자는 요청 시 개별 대시보드의 <em>뷰</em> 제한을 설정할 수 있습니다. 이 기능을 활성화하려면 해당 계정팀 또는 <a href="/help/">Datadog 지원팀</a>에 문의하세요.</div>

{{< img src="dashboards/access_popup.png" alt="사용자가 대시보드에 액세스할 역할을 선택하도록 하는 드롭다운 메뉴가 있는 대화 상자입니다." style="width:70%;">}}

세분화된 액세스 제어를 사용하여 특정 대시보드를 편집할 수 있는 [역할][8]을 제한합니다.
1. 대시보드를 보는 동안 오른쪽 상단에 있는 톱니바퀴 모양 **Configure**을 클릭합니다.
1. **권한**을 선택하세요.
1. **액세스 제한**을 클릭하세요.
1. 대화 상자가 업데이트되어 기본적으로 **뷰어** 액세스 권한이 있는 조직 구성원을 표시합니다.
1. 드롭다운을 사용해 대시보드를 편집할 수 있는 하나 이상의 역할, 팀, 사용자를 선택하세요.
1. **추가**를 클릭합니다.
1. 대화상자가 업데이트되어 선택한 역할에 **편집자** 권한이 있는지 표시합니다.
1. **저장**을 클릭합니다.

**참고:** 대시보드에 대한 편집 권한을 유지하려면 저장하기 전에 사용자가 소속된 역할을 하나 이상 포함해야 합니다. 역할에 관한 자세한 내용은 [RBAC 문서][8]를 참고하세요.

제한된 액세스가 설정된 대시보드를 복원하려면 아래 단계를 따르세요.
1. 대시보드를 보는 동안 오른쪽 상단에 있는 톱니바퀴 모양 **Configure**을 클릭합니다.
1. **권한**을 선택하세요.
1. **Restore Full Access**를 클릭하세요.
1. **저장**을 클릭합니다.

지원 중단된 "읽기 전용" 설정으로 대시보드를 만든 경우 액세스 제어 목록은 액세스 관리(`user_access_manage`) 권한이 있는 역할 목록으로 미리 채워집니다.

Terraform으로 대시보드를 관리하는 경우, 최신 버전의 Datadog Terraform 공급자를 사용하여 대시보드를 편집할 수 있는 역할을 관리할 수 있습니다. 자세한 내용은 [Terraform Dashboard 역할 제한 가이드][9]를 참고하세요.

액세스 표시기가 각 편집 제한 대시보드 오른쪽 상단에 표시됩니다. 사용자 권한에 따라 **편집 액세스 얻기** 또는 **편집 액세스 요청**이 나타날 수 있습니다. 액세스 표시기를 클릭해 액세스 권한과 대시보드 편집에 필요한 단계를 알아보세요.
