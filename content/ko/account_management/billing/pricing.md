Datadog에는 요구에 맞는 다양한 가격 플랜이 있습니다. 자세한 내용은 [가격][1] 페이지를 참고하세요. 주문에 별도로 명시되지 않는 한 매월 제품 사용량을 기준으로 수수료를 계산합니다. 가장 일반적인 가격 단위는 다음과 같습니다.

## 인프라스트럭처 모니터링

* **호스트**는 물리 또는 가상의 운영 체제 인스턴스입니다. Datadog에서는 매 시간 인프라스트럭처 서비스에서 모니터링하고 있는 고유 호스트 수를 기록합니다.
  * HWMP(하이 워터마크 플랜)에서는 이러한 시간별 측정값을 월말에 가장 높은 것에서 가장 낮은 것 순으로 정렬하며, Datadog은 9번째로 높은 측정값을 기준으로 요금을 청구합니다. 2월은 예외로, 8번째로 높은 측정값을 기준으로 요금을 청구합니다.
  * MHP(하이브리드 월별/시간별 요금제)에서는 최소 월별 약정 금액을 청구하고, 해당 약정을 초과하는 호스트 시간에 대해 시간당 요금을 청구합니다.
* **컨테이너**는 애플리케이션 소프트웨어와 제한된 운영 체제 라이브러리 및 설정을 포함하는 독립형 운영 환경입니다. Datadog에서는 5분마다 한 번씩 Datadog 인프라스트럭처 서비스에서 모니터링하고 있는 고유 컨테이너 수를 기록합니다. Datadog은 모니터링되는 컨테이너의 일부 시간을 기준으로 매월 요금을 청구합니다.
* [**커스텀 메트릭**][2]은 메트릭 이름, 호스트 ID, 태그의 고유한 단일 조합입니다. Datadog 요금은 시간당 Datadog 인프라스트럭처 서비스에 제출된 고유한 커스텀 메트릭의 월 평균을 기준으로 청구됩니다.
* **장치**는 프레임에 하나 이상의 단일 보드 컴퓨터로 구성된 물리적 센서입니다. Datadog에서는 Datadog 인프라스트럭처 서비스에서 동시에 모니터링하는 장치 및 호스트 수를 기록하고 요금을 청구합니다.
* AWS **Fargate 작업**은 AWS의 ECS 컨테이너 오케스트레이션 플랫폼을 통해 설정된 컨테이너 모음입니다. Datadog은 Datadog 인프라스트럭처(또는 APM) 서비스에서 모니터링하고 있는 작업 인스턴스 수를 5분 간격으로 기록합니다. Datadog은 월말에 간격 기반 측정값을 집계하고, 애플리케이션 실행 및 모니터링된 총 시간을 기준으로 요금을 청구합니다.

## APM

* 호스트([인프라스트럭처 모니터링](#infrastructure-monitoring)에 정의됨)에서 실행되는 애플리케이션이 트레이스를 생성하여 Datadog SaaS 애플리케이션에 제출하는 경우, Datadog에서 해당 호스트를 하나의 **APM 호스트**로 계산합니다.
  * 하이 워터마크 플랜(HWMP)에서는 이러한 시간별 측정값을 월말에 높은 순으로 정렬하며, Datadog은 9번째로 높은 측정값을 기준으로 요금을 청구합니다. 2월은 예외로, 8번째로 높은 측정값을 기준으로 요금을 청구합니다.
  * MHP(하이브리드 월별/시간별 요금제)에서는 최소 월별 약정 금액을 청구하고, 해당 약정을 초과하는 호스트 시간에 시간당 요금을 청구합니다.
* **인덱싱된 스팬**은 스택의 개별 서비스에 대한 개별 요청입니다. Datadog 요금은 Datadog APM 내의 [보존 필터][3]에 의해 인덱싱된 총 스팬을 기준으로 청구됩니다.
* **수집된 스팬**은 스택의 개별 서비스에 대한 개별 요청입니다. Datadog 요금은 Datadog APM에 수집된 총 스팬의 기가바이트 수를 기준으로 청구됩니다.

인덱싱 및 수집된 스팬 볼륨 모두를 제어할 수 있습니다. 자세한 내용은 [트레이스 수집][4] 및 [보존][5] 문서를 참고하세요.

## 데이터베이스 모니터링

* Datadog에서는 Datadog Database Monitoring을 통해 매시간 모니터링하고 있는 고유한 데이터베이스 호스트 수를 기록합니다.
  * 하이 워터마크 플랜(HWMP)에서는 이러한 시간별 측정값을 월말에 높은 순으로 정렬하며, Datadog은 9번째로 높은 측정값을 기준으로 요금을 청구합니다. 2월은 예외로, 8번째로 높은 측정값을 기준으로 요금을 청구합니다.
  * MHP(하이브리드 월별/시간별 요금제)에서 Datadog은 최소 월별 약정 금액을 청구하고, 해당 약정을 초과하는 호스트 시간에 대해 Datadog은 시간당 요금을 청구합니다.
* Datadog에서는 특정 시간에 추적되어 설정된 [정규화된 쿼리][6]의 총 개수를 기준으로 요금을 청구합니다.

## 로그 관리 

* **로그**는 운영 체제, 애플리케이션, 또는 기타 소스에서 생성된 활동에 대한 텍스트 기반 기록입니다. Datadog에서는 Datadog Logs 서비스에 제출된 총 기가바이트 수를 기준으로 수집된 로그에 대해 요금을 청구합니다.
* **로그 이벤트**는 Datadog Logs 서비스에서 색인을 인덱싱하는 로그입니다. 선택한 보존 정책에 지정된 비율로 인덱싱을 위해 제출된 로그 이벤트 백만 개당 Datadog 요금이 청구됩니다.

## Cloud SIEM

* **분석된 로그**는 잠재적인 보안 위협을 탐지하기 위해 분석된 운영 체제, 애플리케이션, 또는 기타 소스에서 생성된 활동에 대한 텍스트 기반 기록입니다. Datadog에서는 Datadog Cloud SIEM 서비스에서 수집 및 분석된 총 기가바이트 수를 기준으로 분석된 로그에 요금을 청구합니다.

## 신서틱 모니터링

* **API 테스트**는 단일 URL의 HTTP 또는 HTTPS 요청입니다. Datadog 신서틱 모니터링 서비스에 실행되는 1만 개의 API 테스트 실행당 Datadog 요금이 청구됩니다. 
* **브라우저 테스트**는 가상화된 웹 브라우저를 사용하여 웹 기반 애플리케이션에서 스크립트로 작성된 사용자 작업 순서를 시뮬레이션하는 것입니다. Datadog 신서틱 모니터링 서비스에 대해 실행된 브라우저 테스트 천 개당 Datadog 요금이 청구됩니다.

## 네트워크 성능 모니터링

* Datadog에서는 Datadog NPM 서비스를 통해 동시에 모니터링하고 있는 **네트워크 성능 모니터링**(NPM) 호스트 수를 시간당 한 번씩 기록합니다.
  * 하이 워터마크 플랜(HWMP)에서는 이러한 시간별 측정값을 월말에 높은 순으로 정렬하며, Datadog은 9번째로 높은 측정값을 기준으로 요금을 청구합니다. 2월은 예외로, 8번째로 높은 측정값을 기준으로 요금을 청구합니다.
* 또한 매월 모든 NPM 호스트에서 사용되는 총 흐름 수를 측정합니다. **흐름**은 5분 동안 측정된 소스(IP:Port)와 대상(IP:Port) 사이에서 주고받은 트래픽의 기록입니다.

## 실제 사용자 모니터링(RUM)

* **세션**은 웹 애플리케이션 상에서 진행되는 사용자 여정입니다. 15분 동안 활동이 없을 때, 또는 활동이 지속된 지 4시간이 지났을 때 세션이 만료됩니다.

* Datadog에서는 리소스 로딩(XHR, 이미지, CSS 파일, JS 스크립트 등), 프런트엔드 오류, 긴 작업 등 중요한 원격 측정과 함께 최종 사용자가 방문한 모든 페이지를 수집합니다. 이 모든 것이 사용자 세션에 포함됩니다. Datadog Real User Monitoring(RUM) 서비스에서 수집된 세션 1,000개당 Datadog 요금이 청구됩니다.

## Continuous Profiler

* Datadog에서는 Datadog Continuous Profiler 서비스를 통해 동시에 모니터링하고 있는 고유한 Continuous Profiler 호스트 수를 시간당 한 번씩 기록합니다.
  * 하이 워터마크 플랜(HWMP)에서는 이러한 시간별 측정값을 월말에 높은 순으로 정렬하며, Datadog은 9번째로 높은 측정값을 기준으로 요금을 청구합니다. 2월은 예외로, 8번째로 높은 측정값을 기준으로 요금을 청구합니다.
  * 각 호스트에는 최대 4개의 프로파일링된 컨테이너가 무료로 허용됩니다. 이를 초과하는 컨테이너의 가격은 컨테이너당 2달러입니다. 
    **참고**: 이 할당량은 모든 호스트에 걸쳐 집계되므로 모든 호스트에 평균적으로 4개의 컨테이너가 있는 경우, 호스트별로 더 많은 컨테이너가 있는 것과 이 요금이 청구되지 않습니다. 
* Datadog은 프로파일링되는 총 컨테이너 수를 측정합니다. 컨테이너는 애플리케이션 소프트웨어와 제한된 운영 체제 라이브러리 및 설정을 포함하는 자체 완성된 운영 환경입니다. Datadog은 5분마다 한 번씩 Datadog Continuous Profiler 서비스에서 모니터링하고 있는 고유 컨테이너 수를 기록합니다. Datadog은 모니터링되는 컨테이너의 일부 시간을 기준으로 매월 요금을 청구합니다. Continuous Profiler의 경우, Datadog은 모니터링되는 총 컨테이너 수에 대해 Continuous Profiler 서비스를 실행하는 컨테이너만 계산합니다.

## Incident Management

* Datadog은 사고 관리 및 대응에 참여하는 월별 활성 사용자 수를 추적합니다.
 * **활성 사용자**는 인시던트의 댓글이나 신호(그래프, 링크 등)를 제공한 경우에만 계산됩니다. 인시던트를 열고 닫거나 혹은 인시던트를 보기만 한 사용자는 포함되지 않습니다. 또한 이는 명명된 자리 아니므로 액세스 권한이 있는 특정 사용자를 결정할 필요가 없습니다.

## CI Visibility

* Datadog에서는 CI Visibility 서비스로 테스트 및 파이프라인 데이터를 보내는 고유 커밋 사용자 수를 추적합니다.
* **커미터**는 git 작성자 이메일 주소로 식별되는 활성 git 커미터를 의미합니다. 커미터가 특정 달에 최소 3번 커밋하면 요금 청구에 포함됩니다.
  * 파이프라인이 git 저장소와 연결되어 있지 않거나 git 메타데이터를 사용할 수 없는 이벤트의 경우, 파이프라인 실행을 트리거하는 사람의 사용자 이름이 청구 가능한 커미터로 사용됩니다.
* Pipeline Visibility의 경우 모든 파이프라인, 파이프라인 단계 및 파이프라인 작업은 **파이프라인 스팬**으로 계산됩니다. 테스트 가시성을 위해 모든 개별 테스트 실행은 **테스트 스팬**으로 계산됩니다.
