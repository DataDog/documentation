1. ドロップダウンのオプションを 1 つ選択し、パイプラインで予想されるログの量を入力します。
|   オプション   | 説明 | 
| ---------- | ----------- | 
| Unsure | ログの量を予測できない場合や、Worker をテストしたい場合にこのオプションを使用します。このオプションは、最大 2 つの汎用 `t4g.large` インスタンスで EC2 Auto Scaling グループをプロビジョニングします。| 
| 1-5 TB/day | このオプションは、最大 2 つのコンピュート最適化インスタンスで EC2 Auto Scaling グループをプロビジョニングします。`c6g.large`|
| 5-10 TB/day | このオプションは、最小 2 つ、最大 5 つのコンピュート最適化 `c6g.large` インスタンスで EC2 Auto Scaling グループをプロビジョニングします。|
| >10 TB/day | Datadog では、大規模なプロダクションデプロイメントにこのオプションを使用することを推奨しています。このオプションは、最小 2 つ、最大 10 のコンピュート最適化 `c6g.xlarge` インスタンスで EC2 Auto Scaling グループをプロビジョニングします。|
   
   **注**: その他のパラメーターはすべて、Worker デプロイメントに適したデフォルト値に設定されていますが、スタックを作成する前に必要に応じて AWS Console でユースケースに合わせて調整できます。
1. Worker のインストールに使用する AWS リージョンを選択します。
1. [**Select API key**] をクリックして、使用する Datadog API キーを選択します。
1. [**Launch CloudFormation Template**] をクリックして AWS Console に移動し、スタックコンフィギュレーションを確認してから起動します。CloudFormation のパラメーターが期待通りであることを確認します。
1. Worker のインストールに使用する VPC とサブネットを選択します。
1. IAM の必要な権限を確認して、権限のチェックボックスをオンにします。[**Submit**] をクリックしてスタックを作成します。CloudFormation はこの時点でインストールを処理します。Worker インスタンスが起動し、必要なソフトウェアがダウンロードされ、Worker が自動的に開始されます。
1. Observability Pipelines のインストールページに戻り、[**Deploy**] をクリックします。

パイプラインのコンフィギュレーションを変更したい場合は、[Update Existing Pipelines][7001]を参照してください。
